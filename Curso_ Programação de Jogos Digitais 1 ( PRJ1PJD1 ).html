Vamos tratar as colisões

No Unity existem 2 tipos de colisões :

Colisão baseada em física e Colisão desencateada em eventos. Nessa situação a colisão não será física, pois quando esses dois objetos se tocarem você apenas deseja que o power-up seja coletado pela nave do jogador. Agora se fosse em um ambiente baseado em física, quando esses dois objetos colidissem muitas coisa poderia acontecer, como por exemplo, os objeto poderia explodir, ou eles poderiam quicar e cada objeto ter a sua trajetória alterada para uma posição diferente, mas não é isso que você tem que fazer, na verdade você terá que criar uma forma de transformar o powerup em um item coletável, ou seja, quando o Player colidir com o power-up ele será coletado por ele, isso significa que o power-up será destruído no momento da colisão e dará o seu poder para a nave do jogador. 

Uma colisão só acontece quando um objeto toca no outro, no entanto, se esses objetos não possuírem um componente de colisão não tem como o Unity sabe que um objeto colidiu com o outro.

Com o objeto Player selecionado clique no botão Add Component no painel Inspector.


Você escolheu o componente Box Collider 2D porque está criando uma área de colisão em um objeto que faz parte de um jogo 2D, se o jogo fosse em 3D seria o componente Box Collider.

Observe que mesmo que os objetos tenham um componente de colisão anexado a eles, nada acontece. Isso porque, para que ocorra uma colisão, é preciso que exista algo que faça com que haja o contato físico entre esses objetos. Para que haja esse contato físico é preciso que pelo menos um dos objetos que possua um componente colisor tenha também um componente que dê a esse objeto um corpo tangível, é para isso que serve um componente chamado Rigidbody. O Rigidbody é o que permite que a física funcione dentro do Unity. Assim como os componentes Box e Circle Collider, existe um componente Rigidbody tanto para objetos 2D como para 3D.

Contudo, esse não é o comportamento que você deseja que ocorra. O que de fato precisa acontecer é que quando o objeto PU-DisparoTriplo colidir com o objeto Player, é que seja verificado apenas se os colisores de ambos os objetos então um sobre o outro e que a colisão não ocorra de fato e para fazer isso você precisa usar um recurso chamado de Collision Trigger ou gatilho de colisão.

Um gatilho de colisão não executa de fato a colisão entre objetos que possui um corpo rígido. Em vez disso, ele é disparado quando a área de colisão de objeto toca na área de colisão de outro objeto enviando assim uma mensagem informado que tal colisão ocorreu. CRIANDO POWER-UPS PARA O JOGO ( TIRO TRIPLO )

O que é um Power up?

Power up é um termo muito usado em jogos de videogame/computador para designar certos objetos (itens) encontrados pelo cenário, os quais ao serem coletados proporcionam, ao personagem que o pegou, uma vantagem dentro do jogo. Existem diversos meios para se conseguir um Power up, sendo que a sua função ou a maneira pela qual ele será adquirido variará dependendo do estilo do jogo, podendo variar desde coletar letras pelo cenário, até ter que matar determinado inimigo.

No nosso jogo , Como os power-ups irão funcionar dentro do jogo? Bem, a ideia é que: 

  • Cada power-up será um objeto independente dentro do projeto; 
  • Eles serão gerados aleatoriamente no jogo;
  • Eles irão aparecer de cima para baixo, e por essa razão é preciso criar um comportamento que faça essa movimentação;
  • Eles poderão ser coletados apenas pelo objeto Player, o que significa que será preciso trabalhar com colisões.
